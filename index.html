Projects

AATracker.net

A web application to track America's Army 2.5 player stats written using ASP.NET Core, React, C# with .NET 7. Hosted using Nginx on CentOS 7 (service) & Ubuntu (web app). I work with the maintainers of America's Army 2.5 to provide data which allows them to update players' levels in-game.

Servers are queried by sending a UDP request using the GameSpy V2 protocol to the servers query port. The response (in the form of a byte array) is parsed and used to track individual player stats. TLS secured gRPC is used for communication between the web app and the tracking service.

This is my most recent project, beginning development in 2023 alongside the tracking software which continuously monitors the tracked game servers. I intend to open-source some of this project eventually. For the time being, I am more than willing to discuss and review the source!

Tracker UDP Query Library

Class library used to query servers using the GameSpy V2 protocol used by both tracking webapp and service. Makes use of both Polly (for resiliency) and Fast-Member (access type properties by name known only at runtime).

ASP.NET Web API

Straight forward ASP.NET Web API which was started for a technical assessment. Showcases Entity Framework with Sqlite, dynamic dependency injection, generics, delegates, async/await, etc.

SCUM Overlay

The SCUM Server Status Overlay is a project I began in 2018 to allow a friend of mine to monitor server status in-game as they owned a single monitor. It is a very simple application that queries the BatlleMetrics API to retrieve game server stats and display them not only in a Winforms GUI, but also an in-game overlay.
